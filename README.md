# GLR-Net

> [!IMPORTANT]

湖泊水体分割模型，采用 Global + Patch 两阶段训练与预测流程。

A lake water segmentation model based on a Global + Patch two-stage training and prediction workflow.

论文 / Paper: [A Global–Local Residual Refinement Framework for Accurate Lake Boundary Delineation in Remote Sensing Imagery](https://doi.org/10.3390/rs18121919)

发表于 / Published in *Remote Sensing*, 2026, 18(12), 1919。

欢迎围绕湖泊边界提取、遥感语义分割和相关方向开展学术合作。

For detailed model descriptions and design choices, please refer to the paper.
Questions are welcome via GitHub Issues or email, and academic collaboration is encouraged.

## 数据预处理 / Data Preprocessing

输入文件：

Input files:

```text
Lake_sentinel2_GEE/{lake_id}.tif  # 4-band Sentinel-2 DN
Lake_Mask/{lake_id}.tif           # Water mask (0/1)
```

运行 / Run:

```bash
python preprocess/build_dataset_from_tiff.py
```

默认输出至 `DataSet6/`：

Default output under `DataSet6/`:

```text
DataSet6/
├── Train/
├── Val/
├── Test/
└── dataset_split_manifest.json
```

`Train/train_band_mean_std.txt` 保存训练集归一化统计量。默认随机种子为 `42`，
Train/Val/Test 比例为 `7:1.5:1.5`。

`Train/train_band_mean_std.txt` stores the training-set normalization statistics. The default random seed is `42`, and the Train/Val/Test split ratio is `7:1.5:1.5`.

## 使用流程 / Workflow

1. 构建 NPY 数据集 / Build the NPY dataset:

   ```bash
   python preprocess/build_dataset_from_tiff.py
   ```

2. 训练 Stage1 和 Stage2 / Train Stage1 and Stage2:

   ```bash
   python train.py
   ```

3. 执行 Test 集预测 / Run Test-set prediction:

   ```bash
   python predict_stage2_full.py
   ```

4. 计算 Test 集汇总指标 / Compute overall Test-set metrics:

   ```bash
   python evaluate_stage2_test_metrics.py
   ```

5. 计算论文口径统一指标 / Compute paper-style unified metrics:

   ```bash
   python metrics/recalc_test_metrics_unified.py
   ```

6. 生成逐湖指标表 / Generate the per-lake metric table:

   ```bash
   python evaluate_test_lake_table.py
   ```

## 指标定义 / Metric Definitions

- **Hard IoU**：合并全部 Test 集前景像素后计算 micro IoU。
  **Hard IoU:** micro IoU computed by pooling foreground pixels across the full Test set.
- **Lake-level IoU**：逐湖 hard IoU 的算术平均。
  **Lake-level IoU:** arithmetic mean of per-lake hard IoU values.
- **Boundary F1@3px**：逐湖计算后平均，边界容差为 3 像素。
  **Boundary F1@3px:** per-lake boundary F1 averaged across lakes with a 3-pixel tolerance.
- **BDE**：双向边界距离误差，报告 mean、median 和 P90。
  **BDE:** bidirectional boundary distance error, reported as mean, median, and P90.

详细定义见 `metrics/test_metrics_unified_METHOD.txt`。

Detailed definitions are provided in `metrics/test_metrics_unified_METHOD.txt`.
